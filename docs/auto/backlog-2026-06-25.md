**Problem.** `uwGetSafe` (`src/lib/providers/unusual-whales.ts`) — the safe-fetch path behind most

the Polygon REST funnel, but never for UW).

**Fix (additive, mirrors the existing 5xx branch).**
- New pure, alias-free, unit-testable classifier `src/lib/uw-transient-network.ts` →
  `isUwTransientNetwork(msg)` matching the undici/Node connect-level signatures (and explicitly NOT
  the `→ <status>` HTTP format). Pattern mirrors the sibling `uw-upstream-5xx.ts` module so it loads
  cleanly under `tsx --test`.
- New test `src/lib/uw-transient-network.test.ts` (3 cases: matches the RT-2 signatures; rejects
  429/503/403 HTTP-status messages; rejects unrelated/empty app errors).
- `uwGetSafe`: added a transient-network branch **after** the 5xx branch and **before** the catch-all
  `return null` — bounded exponential backoff retry (default 2 retries), then stale-cache fallback,
  then null. Behind the existing `UW_DEBUG_RETRIES` debug log.

**Safety / all-ifs checked.** Does NOT feed the 429 breaker (connect errors aren't 429; `noteUw429`
never called). Branch order (403 → 429 → 5xx → transient-network → null) means HTTP-status errors are
consumed before the network branch. Bounded by `retries` (≤3 attempts) — no loop. Terminal behavior
when retries exhaust + no stale = `null`, identical to before → strictly better. Scope contained to

### ✅ Implemented → main (`07e0975`): transient-blip retry on the direct-`uwGet` HELIX flow path

**Problem.** `fetchMarketFlowAlertPage` (`src/lib/providers/unusual-whales.ts:578`) — the page fetch
behind `fetchMarketFlowAlertRows`, which feeds the HELIX live flow tape — called **raw `uwGet` with
NO retry**, unlike `uwGetSafe`. A single momentary `api.unusualwhales.com` connect blip (the RT-2
class: `UND_ERR_CONNECT_TIMEOUT` / `EHOSTUNREACH` / `fetch failed` / `ECONNRESET` / DNS hiccup) **or**
an upstream 5xx on the **first** page threw straight to the outer catch; with no fresh-enough stale
`marketFlowCache` (cold process, or cache >30 min old) that served **`[]` — a blank tape** with no
retry attempt.

**Fix (additive, mirrors `uwGetSafe`'s proven branches).** New `uwGetWithTransientRetry<T>()` wrapper
around `uwGet`: bounded exponential-backoff retry (`retries=2`, ≤3 attempts — same ceiling as
`uwGetSafe`) for **ONLY** the transient-network + upstream-5xx classes, reusing the existing
`isUwTransientNetwork` / `isUwUpstream5xx` classifiers (already unit-tested). Wired into
`fetchMarketFlowAlertPage`. Behind the existing `UW_DEBUG_RETRIES` debug log.

**Safety / all-ifs checked.**
- **429 unchanged:** 429 (and every other HTTP-status error) is NOT retried here — it bubbles exactly
  as before, so the flow-alerts breaker accounting (`noteUw429` in both the page-loop catch and the
  outer catch) and the 403/partial-serve semantics are byte-for-byte preserved. Only the no-retry
  transient classes gained a retry.
- **No double-count:** transient-network + 5xx never call `noteUw429`, so the breaker can't be fed by
  this path.
- **Bounded:** ≤3 attempts, ~1s+~2s backoff max — no loop, same latency ceiling as `uwGetSafe`; only
  paid on the transient classes.
- **Partial-merge intact:** on page 2+ an *exhausted* transient blip (or a 429) still throws → the
  existing "keep the pages we have / serve partial" path runs unchanged. Updated that catch's now-stale
  "no internal retry" comment to match.
- Scope contained to `unusual-whales.ts`; `uwGetSafe` and the broader RT-2 SPX-play contract untouched.
