### ✅ Implemented → main (`9b51d88`): signal silent options-chain pagination truncation

**Problem.** `src/lib/providers/polygon-options-gex.ts` has four banded-chain pagination loops, each
bounded by a page-guard cap:
- `fetchHeatmapBand` (cap 16) — the GEX heatmap chain
- `fetchChainBand` (cap 8) — per-expiry GEX walls / desk chain
- `fetchPolygonOiByExpiry` (cap 12) — OI-by-expiry term
- `fetchPolygonIvTermStructure` (cap 20) — IV term structure

Every loop exits when `guard` reaches its cap **without checking whether `page.next_url` is still
set**. So when a chain exceeds the cap, the remaining pages are dropped and the derived GEX walls /
OI / IV-term are **understated — silently, with no signal at all**. That is a data-correctness hole on
a flagship surface (heatmap / SPX desk) that directly violates the standing "every value correct &
grounded" mandate, and the API-integration audit flagged it as `(observability)`.

**Fix (purely additive, observability only — zero value/behavior change).**
- New module-local helper `warnChainTruncated(label, underlying, pages)` → emits ONE
  `console.warn` (`[polygon-gex]` prefix, matching the file's existing warn convention).
- After each of the four loops: `if (page?.next_url) warnChainTruncated(<fn>, <underlying>, guard);`
  — fires **only** on genuine cap-truncation, so truncation is no longer invisible.

**Safety / all-ifs checked.**
- No false positives: a loop exits via `!page` (→ `page?.next_url` is `undefined`, no warn), via the
  clean `if (!page.next_url) break` (→ no next_url, no warn), or via the guard cap (→ page truthy +
  next_url present → warns). The warn fires *only* in the third case.
- No hot-path cost (warn only in the rare truncation case); no extra API calls; no change to returned
  values — magnitudes are byte-identical to before, only the log gains a signal.
- If the warning recurs frequently in prod, that IS the signal the audit wanted (caps too low) — the
  value-changing follow-up (raise caps / fully paginate next_url) remains **flagged**, not auto-done,
  because it changes GEX numbers + upstream cost.
- Brand/grey + secrets: N/A (server provider lib, no UI, no client-exposed values).

**Validation (all green):** `npx tsc --noEmit` exit 0 · `npm run build` exit 0.
