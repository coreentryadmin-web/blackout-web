## P1 — Fix Today (Feature Broken or Degraded)
- **🟠 DATA INTEGRITY — play outcomes mislabel profitable STOP/THESIS exits as losses.**
  `src/lib/spx-play-outcomes.ts:177-178`:
  ```ts
  if (close.was_loss || close.exit_action === "STOP" || close.exit_action === "THESIS") {
    return "loss";   // ← returns "loss" WITHOUT checking pnl_pts sign
  }
  ```
  The THETA/SESSION branch (:170-175) and TRAIL branch (:183-186) both correctly key off
  `pnl_pts`, but STOP/THESIS exits are hard-coded to `loss`. A THESIS exit is frequently a
  **win** (thesis target reached on a winner). Prior run captured live proof: play #3 (long,
  exit_action=THESIS, entry 7432.13 → exit 7439.43, `pnl_pts = +7.30`) is stored
  `outcome='loss'`. **This is the root cause of the "0-for-3" reading below** — the track
  record understates win-rate on every P&L / track-record surface. **Fix (one-liner):** for
  STOP/THESIS, `pnl_pts > 0` → `win` (mirror TRAIL). Carried unfixed across the 16:46, 20:38
  (2026-06-29) runs and this one. Not auto-fixed here: outcome classification is data-integrity
  + behavior-affecting, belongs in a reviewed/verified change, not a cron commit.
